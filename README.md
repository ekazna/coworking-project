# Сервис автоматизации процессов управления коворкингом

Дипломный проект, 2026

Цель проекта — автоматизировать базовые процессы управления коворкингом:

* бронирование рабочих мест и оборудования;
* управление ресурсами и их доступностью;
* обработку обращений пользователей;
* учет поломок и периодов недоступности ресурсов (рабочих мест, оборудования);
* отправку уведомлений пользователям;
* отображение аналитики для администратора.

Разработаны:
 - личный кабинет
 - редактирование профиля
 - страница бронирования, продления бронирования
 - просмотр активных и прошлых броней
 - подача обращения
 - интерфейсы администратора:
   - брони
   - управление поломками
   - управление ресурсами
   - управление клиентами
   - аналитика
  
## Технологический стек

* Python
* Django, Django REST Framework, Django ORM
* PostgreSQL
* React, JavaScript, CSS
* Telegram Bot API

## Основной функционал

### Для пользователя

Пользователь может:

* зарегистрироваться и войти в личный кабинет;
* редактировать профиль;
* просматривать доступные рабочие места и оборудование;
* выбирать дату и время бронирования;
* создавать бронирование;
* продлевать активную бронь;
* просматривать активные и прошлые бронирования;
* отправлять обращение о неисправности;
* получать уведомления о бронированиях, изменениях и поломках.

### Для администратора

Администратор может:

* просматривать список клиентов;
* управлять рабочими местами, оборудованием и другими ресурсами;
* редактировать параметры ресурсов;
* просматривать и модерировать бронирования;
* обрабатывать обращения пользователей;
* фиксировать поломки и выводить ресурсы из эксплуатации;
* управлять периодами недоступности ресурсов.


## Backend-модули

Серверная часть разделена на несколько Django-приложений:

```text
backend/
├── bookings/       # Бронирования, продление, изменение, журнал изменений
├── issues/         # Обращения пользователей и фиксация неисправностей
├── notifications/  # Системные, email- и Telegram-уведомления
├── resources/      # Рабочие места, оборудование, категории и типы ресурсов
├── users/          # Пользователи, профили, роли и аутентификация
├── config/         # Настройки Django-проекта
└── manage.py
```

## Модель данных

Ключевые сущности: 
- Пользователи (User, Profile)
- Ресурсы (Category, Type, Resource)
- Бронирования (Booking, ChangeLog)
- Обращения (Issue, ResourceOutage)
- Уведомления (Notification)

Особенности: 
- поддержка связанных броней
- хранение истории изменений
- учёт недоступности ресурсов

## Логика бронирования

В системе реализована логика проверки доступности ресурсов перед созданием или продлением бронирования.

Учитываются:

* выбранный ресурс;
* дата и время начала бронирования;
* дата и время окончания бронирования;
* рабочие часы коворкинга;
* существующие активные бронирования;
* периоды недоступности ресурса;
* связанные бронирования оборудования;
* статус бронирования.

При продлении бронирования система проверяет, можно ли продлить бронь на выбранном ресурсе без пересечений. Если ресурс занят, система определяет максимально возможное время продления или предлагает альтернативные варианты.

## Обращения и поломки

Пользователь может отправить обращение о неисправности рабочего места или оборудования. После обработки обращения администратор может зафиксировать проблему и вывести ресурс из эксплуатации на определенный период.

В этом случае ресурс исключается из доступных для новых бронирований, а пользователи получают уведомления об изменениях.

## Уведомления

В проекте реализован механизм уведомлений пользователей.

Уведомления могут создаваться при следующих событиях:

* создание бронирования;
* продление бронирования;
* изменение бронирования;
* отмена бронирования;
* фиксация неисправности;
* вывод ресурса из эксплуатации;
* изменение статуса обращения.
